# 节点开发规范

本文档总结了开发过程中摸索出的节点设计规范，新节点开发必须遵循。

---

## 一、节点类型与端口颜色规范

所有端口颜色统一定义在 `portStyles.ts`，禁止在节点内硬编码颜色。

| 端口类型 | 颜色 | 含义 |
|---|---|---|
| `image` | `#f97316`（橙色） | 单张图片 base64 |
| `images` | `#fb923c`（浅橙） | 图片数组（暂未启用，保留） |
| `text` | `#3b82f6`（蓝色） | 单条文本字符串 |
| `texts` | `#60a5fa`（浅蓝） | 文本数组（暂未启用，保留） |

## 二、节点数据结构规范

每个节点导出一个 `interface XxxNodeData`，字段分两类：

```ts
export interface MyNodeData {
  // ── 用户配置字段（持久化）──
  label: string       // 节点显示名，必须有
  someConfig: string  // 其他配置

  // ── 运行时字段（运行后填充，非持久化）──
  outputText?: string   // 运行后显示的预览
  savedCount?: number
}
```

**规则：**
- `label` 必须存在，用于节点标题显示
- 运行后填充的字段用 `?` 可选，命名前缀清晰（`output`、`saved`、`received`）
- `FileSystemDirectoryHandle` 等不可序列化对象标注注释说明

## 三、NodeShell 使用规范

所有节点必须使用 `NodeShell` 作为外壳，**禁止**自定义最外层容器。

```tsx
<NodeShell title={d.label ?? '节点名'} icon={<Icon size={13} />} selected={selected}>
  <NodeResizer ... />  {/* 第一行，紧跟在 NodeShell 内 */}
  {/* 节点内容 */}
</NodeShell>
```

**NodeShell 注意事项：**
- 外层 div **没有** `overflow: hidden`（去掉是为了防止裁剪 Handle）
- 标题栏单独有 `overflow: hidden` 保持圆角
- 内容区 `height: calc(100% - 32px)`，配合 NodeResizer 使用

---

## 四、NodeResizer 规范

每个节点必须加 `NodeResizer`，放在 NodeShell 第一个子元素位置。

```tsx
import { NodeResizer } from '@xyflow/react'

<NodeResizer
  minWidth={200}
  minHeight={120}
  isVisible={selected}
  lineStyle={{ borderColor: 'rgba(255,255,255,0.2)' }}
  handleStyle={{ borderColor: 'rgba(255,255,255,0.3)', background: '#222' }}
/>
```

**各节点建议最小尺寸：**

| 节点 | minWidth | minHeight |
|---|---|---|
| imageUpload | 180 | 140 |
| imageSave | 180 | 120 |
| textInput | 180 | 100 |
| textDisplay | 180 | 80 |
| llm | 240 | 200 |

## 五、Handle 定位规范

### 普通 Handle（固定端口）

直接放在 NodeShell 内，使用 `handleStyle()` 工具函数：

```tsx
import { handleStyle } from './portStyles'

<Handle
  type="source"
  position={Position.Right}
  id="text"
  style={handleStyle('text', 'right')}
  title="text"
/>
```

`handleStyle` 返回 `{ right: -6 }` 或 `{ left: -6 }`，使 Handle 正好突出节点边缘 6px。

### 动态 Handle（LLM 槽位等）

动态 Handle 必须放在 `position: relative` 的父容器内，使用 **手动偏移** 补偿 NodeShell 内边距：

```tsx
// 槽行容器
<div style={{ position: 'relative', ... }}>
  <Handle
    type="target"
    position={Position.Left}
    id={slot.id}
    style={{
      position: 'absolute',
      left: -16,          // -(NodeShell内边距10px + 突出6px) = -16
      top: '50%',
      transform: 'translateY(-50%)',
      background: PORT_COLORS[slot.type === 'image' ? 'image' : 'text'],
      border: '2px solid #0a0a0a',
      width: 11,
      height: 11,
    }}
  />
  {/* 槽内容 */}
</div>
```

**关键：** 动态 Handle 数量变化时，必须调用 `useUpdateNodeInternals` 通知 React Flow 重新测量位置，否则连线不会显示：

```tsx
import { useUpdateNodeInternals } from '@xyflow/react'

const updateNodeInternals = useUpdateNodeInternals()

useEffect(() => {
  updateNodeInternals(id)
}, [id, slots.length, updateNodeInternals])
```

## 六、updateData 规范（避免 stale closure）

**必须**通过 `useAppStore.getState()` 读取最新状态，不能直接用 `useCallback` 闭包里的 state：

```tsx
// ✅ 正确
const updateData = useCallback((patch: Partial<MyNodeData>) => {
  const nodes = useAppStore.getState().rfNodes
  useAppStore.getState().setRfNodes(
    nodes.map((n) => n.id === id ? { ...n, data: { ...n.data, ...patch } } : n)
  )
}, [id])

// ❌ 错误：rfNodes 是闭包里的旧值
const updateData = useCallback((patch) => {
  setRfNodes(rfNodes.map(...))
}, [rfNodes])
```

## 七、键盘事件拦截规范

所有 `input` 和 `textarea` 必须阻止键盘事件冒泡，防止用户在输入框打字时误触发节点删除（Backspace）：

```tsx
const stopPropagation = (e: React.KeyboardEvent) => e.stopPropagation()

<input onKeyDown={stopPropagation} ... />
<textarea onKeyDown={stopPropagation} ... />
```

## 八、注册新节点的完整清单

每新增一个节点，需要修改以下 **4 个位置**，缺一不可：

### 1. `WorkflowView/index.tsx` — NODE_TYPES

```ts
const NODE_TYPES = {
  // ...已有节点
  myNode: MyNode,
}
```

### 2. `WorkflowView/index.tsx` — NODE_DEFAULTS

```ts
const NODE_DEFAULTS: Record<string, Record<string, unknown>> = {
  // ...已有节点
  myNode: { label: '我的节点', someConfig: 'default' },
}
```

### 3. `WorkflowView/index.tsx` — NODE_SIZES

```ts
const NODE_SIZES: Record<string, { width: number; height: number }> = {
  // ...已有节点
  myNode: { width: 240, height: 180 },
}
```

### 4. `CanvasContextMenu.tsx` — NODE_MENU_ITEMS

```ts
export const NODE_MENU_ITEMS: MenuItem[] = [
  // ...已有节点
  {
    type: 'myNode',
    label: '我的节点',
    icon: <MyIcon size={13} />,
    description: '节点功能描述',
  },
]
```

### 5. `DAGEngine.ts` — 执行器

在 `switch (node.type)` 中添加对应 case，并实现 `execMyNode` 函数：

```ts
case 'myNode':
  output = await execMyNode(node, upstream, patchNodeData)
  break
```

## 九、DAGEngine 节点间数据传递规范

节点间通过 `NodeOutput` 传递数据：

```ts
interface NodeOutput {
  images?: string[]   // base64 字符串数组（不含 data: 前缀）
  text?: string       // 文本字符串
}
```

**规则：**
- `execXxx` 函数从 `upstream` 读取上游数据，返回本节点的输出
- 多个上游节点的 `images` 会自动合并为数组，`text` 取最后一个
- 需要更新节点卡片显示时，通过 `patchNodeData(node.id, { ... })` 写回

## 十、File System Access API 类型声明

`showOpenFilePicker`、`showDirectoryPicker` 等 API 在 TypeScript 中类型不完整，需强制转型：

```ts
// ✅ 用 (window as any) 绕过类型检查
const [handle] = await (window as any).showOpenFilePicker({ ... })
const dirHandle = await (window as any).showDirectoryPicker({ mode: 'readwrite' })
```
