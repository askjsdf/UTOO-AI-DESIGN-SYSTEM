## 一、产品背景与目标

### 1.1 背景

UTOO DESIGN 在香水包装设计业务中，每一个设计方案在交付客户前需要生成四类配套视觉资产：主视觉（KV）、细节矩阵、情绪板、CMF 工艺变体。目前这四类资产由四个独立的 AI Studio 原型工具分别生成，流程割裂、需要手动切换工具、无法批量处理、输出结果散落在不同地方，严重影响设计团队效率。

### 1.2 产品目标

构建一个本地桌面应用，将四条出图工作流整合为统一平台，实现：

- **一张图进来，全套资产自动输出**
- **工作流可视化配置，提示词可直接编辑**
- **批量自动处理，支持夜间无人值守运行**
- **方案结果以设计画布形式呈现，而非传统文件夹**

### 1.3 成功标准

| 指标 | 目标值 |
|------|--------|
| 单张图全套出图（13张）时间 | ≤ 15 分钟 |
| 100 张批量任务完成率 | ≥ 95%（剩余自动重试） |
| 工作流节点提示词修改操作步骤 | ≤ 3 步 |
| 系统崩溃导致任务全部丢失概率 | 0%（持久化队列） |

## 四、模块一：工作流编排器

### 4.1 功能概述

以可视化节点图的方式定义 AI 出图管道。设计师通过拖拽节点、连线、编辑节点参数来构建工作流，而不是修改代码。

### 4.2 节点类型

系统内置五种节点，覆盖所有出图场景：

#### 4.2.1 图片输入节点（Image Input Node）
- 支持单张上传、多张上传、从本地文件夹选取
- 支持拖拽图片到节点
- 显示图片缩略图预览
- 支持来自方案画布的反向输入（见第五章）

#### 4.2.2 分析节点（Analyze Node）
- 调用 Gemini 文本模型对输入图片进行结构化分析
- **可配置项：**
  - 系统提示词（在节点内直接编辑，支持多行）
  - 输出 JSON Schema（定义期望的结构化输出字段）
  - 模型选择（gemini-3.1-pro-preview 等）
- 输出：结构化 JSON 分析结果（Visual DNA、风格分析、情绪板轴向等）
- 分析结果可被下游多个节点同时复用，避免重复调用

#### 4.2.3 提示词生成节点（Prompt Node）
- 将分析结果转换为图像生成指令
- **可配置项：**
  - 提示词模板（支持 `{{变量}}` 插值语法引用上游分析结果）
  - 生成数量（1-8个）
  - 输出格式（单个字符串 / 字符串数组）
- 可选：接收参考图作为额外输入

#### 4.2.4 图像渲染节点（Render Node）
- 调用 Gemini 图像生成模型生图
- **可配置项：**
  - 主模型（gemini-3.1-flash-image-preview 等）
  - 降级备用模型（主模型失败时自动切换）
  - 宽高比（16:9 / 1:1 / 9:16）
  - 输出分辨率（1K / 4K）
  - 节点内并发数（1-4，控制同时发起几张图的生成）
- 输出：图片文件（存本地磁盘）

#### 4.2.5 输出节点（Output Node）
- 收集上游所有渲染节点的输出
- 将图片推送到方案画布（指定画布区域坐标）
- 触发本地文件保存（按项目目录结构归档）

### 4.3 预置工作流模板

系统内置四个预置工作流，对应原有四个工具的能力，开箱即用：

| 模板名称 | 对应原工具 | 输出 |
|---------|-----------|------|
| KV 主视觉 | Nocturne KV v5.1 | 4 张 16:9 场景图 |
| 细节矩阵 | Detail Matrix Engine | 1 张画中画矩阵图 |
| 情绪板 | Intellectual Moodboard Engine | 4 张多面板情绪板 |
| CMF 工艺变体 | CMF Narrative Generator | 4 张×5变体组合图 |

**全套一键工作流**（核心需求）：将四个模板并联，单次输入触发全部四条管道同步执行，自动输出全套 13 张图。

### 4.4 工作流管理

- 工作流以 JSON 文件保存在本地，可导入/导出/分享
- 支持复制已有工作流并修改（避免从头配置）
- 支持工作流命名和描述
- 历史运行记录可追溯（哪个工作流、哪张图、什么时间、结果如何）

### 4.5 运行控制

- 单张图运行：选中输入图，选择工作流，点击运行
- 批量运行：选择多张图或文件夹，进入批量队列
- 运行时显示逐节点进度（分析中 / 生成第 X/Y 张 / 完成）
- 支持暂停、继续、取消单个任务

## 五、模块二：方案画布

### 5.1 功能概述

以无限画布的形式展示所有生成结果，设计师可以在画布上自由排列、标注、分组图片，并可从画布直接触发新的工作流（反向操作）。

### 5.2 画布基础能力

- 无限画布，支持鼠标/触控板缩放和平移
- 生成的图片以卡片形式自动落位到画布
- 图片卡片支持：拖动、缩放、旋转
- 支持 Frame（分组框）：将一组图片框起来并命名
- 支持文字便签：在画布上添加设计评注
- 支持连接箭头：标注图片之间的演化关系
- 小地图（MiniMap）辅助大画布导航

### 5.3 图片自动布局

每次完整运行后，13 张输出图自动按区域排布在画布上：

```
每个"项目区域"的自动布局：

  ┌─────────────────────────────────┐
  │  Frame: KV 主视觉               │
  │  [KV1] [KV2] [KV3] [KV4]      │
  └─────────────────────────────────┘

  ┌─────────────────────────────────┐
  │  Frame: 情绪板                  │
  │  [M1]  [M2]  [M3]  [M4]       │
  └─────────────────────────────────┘

  ┌────────────────┐  ┌─────────────────────────┐
  │  Frame: 矩阵   │  │  Frame: CMF 变体         │
  │  [Matrix]      │  │  [P1] [P2] [P3] [P4]   │
  └────────────────┘  └─────────────────────────┘
```

多个项目（多张输入图的结果）在画布上横向排列，形成方案矩阵。

### 5.4 图片卡片信息

每张图片卡片上显示：
- 图片缩略图
- 类型标签（KV / 矩阵 / 情绪板 / CMF）
- 生成时间
- 来源工作流名称
- 悬停时显示完整尺寸预览

### 5.5 反向触发工作流（双向交互核心需求）

**需求描述：** 在画布上选中一张已生成的图片，可以将其作为输入，触发新的工作流，生成的新图片自动出现在原图旁边，并用箭头标注演化关系。

**典型使用场景：**
- 方案图颜色不合适 → 选中图片 → 触发"修改 CMF"工作流 → 新配色版本出现在旁边
- 某张 KV 图构图很好 → 选中图片 → 触发"生成细节矩阵"工作流 → 矩阵图出现在旁边
- 某张情绪板需要延伸 → 触发自定义工作流 → 结果接续排列

**交互方式（第一版）：** 右键点击画布上的图片卡片，弹出菜单：
```
  ├── 下载图片
  ├── 复制到剪贴板
  ├── 用工作流处理...
  │     ├── 修改 CMF 工艺
  │     ├── 重新生成 KV
  │     ├── 生成细节矩阵
  │     └── 选择其他工作流...
  └── 删除
```

**空间关联：** 反向工作流输出的新图片自动放置在源图右侧（间距 60px），并用箭头连接，保留设计演化的脉络。

### 5.6 项目管理

- 画布状态（所有图片位置、Frame、便签）持久化保存为项目文件
- 支持创建多个项目（如：按客户、按产品线区分）
- 项目列表页展示所有历史项目，可按时间/名称排序
- 支持打开历史项目，完整还原上次画布状态

### 5.7 导出功能

| 导出类型 | 说明 |
|---------|------|
| 单张图片 | 下载原始分辨率图片 |
| 按 Frame 导出 | 导出某一类型的所有图片（如只导出 KV 图）|
| 全项目 ZIP | 按目录结构打包全套图片 + 元数据 |
| 画布截图 | 导出当前画布视图为一张大图（适合给客户快速预览）|

## 六、模块三：自动化引擎

### 6.1 功能概述

支持大批量图片的无人值守自动处理，具备任务调度、进度监控、故障恢复能力。

### 6.2 批量输入方式

#### 6.2.1 文件夹监听模式
- 在设置中指定一个本地"监听文件夹"（如 `~/UTOO/input/`）
- 开启监听后，向该文件夹放入图片，系统自动检测并创建任务
- 支持指定该模式下默认使用的工作流（如：全套工作流）
- 新图入队后，若 Worker 空闲则立即开始；否则排队等待

#### 6.2.2 定时任务模式
- 在 UI 中设置定时规则（如：每天 23:00、每周一 8:00）
- 触发时扫描监听文件夹中所有未处理的图片，批量创建任务
- 支持一次性定时和循环定时两种模式
- 适合夜间批量处理：白天整理好图片，晚上自动跑完

#### 6.2.3 手动批量模式
- 在 UI 中手动选择多张图片或整个文件夹
- 选择目标工作流，点击"开始批量运行"
- 任务进入队列，可实时查看进度

### 6.3 任务队列

- 任务持久化存储在本地数据库（SQLite），应用重启后队列不丢失
- 任务状态：待处理 / 运行中 / 已完成 / 失败 / 重试中 / 已放弃
- 支持暂停/恢复整个队列
- 支持删除单个待处理任务
- 支持优先级设置（插队处理紧急任务）

### 6.4 并发控制

- 默认同时运行 2 个 Worker（每个 Worker 处理一张图的完整工作流）
- 可在设置中调整 Worker 数量（1-4）
- 每个 Worker 内部，四条生成管道（KV/矩阵/情绪板/CMF）并行执行
- API 调用间隔最小 500ms，防止触发 Gemini 限流（429）
- 检测到限流错误（429）时，自动暂停 60 秒后恢复

### 6.5 心跳监控与故障恢复

**心跳机制：**
- 每个运行中的任务每 5 秒更新一次心跳时间戳
- 监控进程每 10 秒扫描一次所有运行中的任务
- 若某任务超过 60 秒未更新心跳，判定为"卡死"，自动重新入队

**重试策略（指数退避）：**

| 重试次数 | 等待时间 | 说明 |
|---------|---------|------|
| 第 1 次失败 | 30 秒后重试 | 临时网络波动 |
| 第 2 次失败 | 2 分钟后重试 | API 服务波动 |
| 第 3 次失败 | 10 分钟后重试 | 持续异常 |
| 第 4 次失败 | 标记为已放弃 | 记录错误，跳过 |

**断点续跑：**
- 分析阶段完成后，结果缓存在任务记录中
- 若任务在生成阶段失败，重试时直接复用分析结果
- 跳过已成功生成的子步骤（如 KV 已生成 2/4 张，重试从第 3 张开始）

### 6.6 批量任务监控面板

**实时状态大盘：**
```
总进度：47/100    预计剩余：3小时 20分    [暂停全部] [停止全部]

运行中 (2)
  · rose_bottle_01.png      KV 生成 3/4     ████░  75%
  · jasmine_concept.png     分析中          ██░░░  40%

重试中 (1)
  · woody_texture.png       等待重试 1分30秒后（第2次）

已完成 (45)                                    [查看全部]
已放弃 (1)                                     [查看原因]
待处理 (51)
```

**任务详情：** 点击任意任务查看：
- 每个节点的执行状态和耗时
- 错误信息和重试历史
- 已生成的图片预览
- 手动重新触发按钮

### 6.7 后台运行

- 关闭主窗口后，应用最小化到系统托盘（Mac 菜单栏 / Windows 任务栏）
- 托盘图标实时显示处理进度（如：`🔄 47/100`）
- 全部完成后：托盘图标变绿 + 系统通知提醒
- 点击托盘图标重新打开主窗口，查看结果

## 十、非功能需求

### 10.1 性能
- 画布加载 1000 张图片时，滚动和缩放帧率 ≥ 30fps
- 单个节点配置更改即时生效，无需重启
- 任务状态更新延迟 ≤ 1 秒（UI 实时反映）

### 10.2 稳定性
- 任何单个任务失败不影响其他任务
- 应用崩溃重启后，队列状态完整恢复
- API 调用失败率 ≤ 5% 时，系统自动恢复无需人工干预

### 10.3 易用性
- 首次安装：输入 API Key 后即可使用，无其他配置门槛
- 工作流模板开箱即用，不要求用户理解 AI 原理
- 所有操作界面语言：中文
- 错误提示使用对设计师友好的描述，不显示技术堆栈信息

### 10.4 数据安全
- API Key 本地加密存储，不上传任何服务器
- 所有图片数据仅存在本地，不传输到第三方（Gemini API 调用除外）
- 不需要用户注册账号

## 十一、后续迭代方向（V2 规划）

以下功能不在第一版范围内，但架构设计时需预留扩展空间：

1. **自定义节点类型**：允许用户编写自定义节点（接入其他 AI API 或本地模型）
2. **工作流市场**：团队内部分享和导入工作流配置
3. **多人协作**：局域网内多台电脑共享画布和任务队列
4. **画布跨设备拖拽**：将图片从 tldraw 画布直接拖入 React Flow 输入节点（替代右键菜单的更直觉操作）
5. **视频/动效输出**：支持生成动态版本的 KV 主视觉
6. **客户端分享**：将特定画布区域生成只读分享链接发给客户
