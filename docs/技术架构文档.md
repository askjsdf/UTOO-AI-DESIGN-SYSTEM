## 一、技术选型总览

### 1.1 选型决策表

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|---------|
| 运行环境 | **浏览器（Chrome/Edge）** | 最新版 | 直接运行，无需安装，Mac/Windows 通用；File System Access API 在现代 Chrome 完全支持 |
| 构建工具 | **Vite** | 6+ | 与现有四个原型同栈，热更新快，零配置 React 集成 |
| 前端框架 | **React + TypeScript** | 19 / 5+ | 与现有原型同栈，两个画布库均为 React 原生支持 |
| 工作流画布 | **@xyflow/react** | 12+ | MIT 开源，只做画布 UI 不干预业务逻辑，自定义节点完全自由 |
| 方案画布 | **@tldraw/tldraw** | 3+ | 专业无限画布 SDK，支持自定义 Shape 类型，编程式 API 完整 |
| 全局状态 | **Zustand** | 5+ | 轻量，无 Redux 样板代码，支持 subscribe 实现跨组件事件总线 |
| AI 服务 | **@google/genai** | latest | 现有原型已验证，直接在浏览器端调用 Gemini API |
| 文件系统 | **File System Access API** | 浏览器原生 | showDirectoryPicker / showOpenFilePicker / showSaveFilePicker，Chrome/Edge 完全支持 |
| 配置存储 | **localStorage / IndexedDB** | 浏览器原生 | API Key 和用户设置持久化，无需 electron-store |
| 并发控制 | **p-queue** | latest | API 调用间隔节流，全局并发上限控制 |
| 图片下载 | **浏览器 download API** | 原生 | base64 → Blob → 下载，或通过 File System Access API 直写磁盘 |

### 1.2 明确排除的技术

| 技术 | 排除原因 |
|------|---------|
| Electron | 打包分发复杂，主进程 CJS/ESM 兼容问题持续，本地工作室场景无需桌面壳 |
| Tauri | 同 Electron，且 WebView 差异风险更大 |
| Next.js | SSR 无意义，增加复杂度 |
| Firebase | 引入云端依赖，本产品定位纯本地 |
| better-sqlite3 | 浏览器不可用，Phase 4 批量队列改用 IndexedDB |
| electron-store | 浏览器不可用，替换为 localStorage |
| node-cron / chokidar | 浏览器不可用，Phase 4 批量调度改用 Web Worker + setInterval |

### 2.2 进程/模块职责划分

**GeminiService**
- Gemini API 调用（analyzeDesign + generateImage）
- 模型降级逻辑
- p-queue 并发控制和限流

**FileService**
- 使用 File System Access API 读取本地图片（showOpenFilePicker）
- 选择输出文件夹（showDirectoryPicker，保持句柄跨页面持久化）
- 将 base64 图片写入本地文件（FileSystemWritableFileStream）

**SettingsService**
- API Key 存储（localStorage，加密可选）
- 用户偏好设置读写

**DAG 执行引擎**
- 工作流图解析 + 拓扑排序
- 节点执行器调度（全部在浏览器主线程 / Web Worker）
- 进度事件通过 Zustand 推送给 UI

## 四、工作流编排器架构

### 4.1 节点类型

| 节点类型 | 职责 | 输入 | 输出 |
|---------|------|------|------|
| **ImageInputNode** | 上传/选择本地图片，转为 base64 | - | image_base64 |
| **AnalyzeNode** | 调用 Gemini 分析香水瓶，返回结构化 JSON | image_base64 | analysis_json |
| **PromptNode** | 用分析结果渲染提示词模板 | analysis_json | prompt_text[] |
| **RenderNode** | 并发调用 Gemini 图像生成 | prompt_text[] | image_base64[] |
| **OutputNode** | 将图片写入本地磁盘 | image_base64[] | file_path[] |

### 4.2 DAG 执行流程

```
ImageInputNode
    │ base64
    ▼
AnalyzeNode ──── Gemini analyzeDesign() ────▶ analysis JSON
    │
    ├──▶ PromptNode(KV)    ──▶ RenderNode(KV)    ──▶ OutputNode
    ├──▶ PromptNode(Matrix) ──▶ RenderNode(Matrix) ──▶ OutputNode
    ├──▶ PromptNode(Mood)  ──▶ RenderNode(Mood)  ──▶ OutputNode
    └──▶ PromptNode(CMF)   ──▶ RenderNode(CMF)   ──▶ OutputNode
```

四条并行分支共享同一个 AnalyzeNode 输出，通过拓扑排序确保分析完成后四支同时启动。

### 4.3 执行引擎核心

```typescript
class DAGEngine {
  // 将 RF nodes+edges 解析为邻接表
  parseGraph(nodes: Node[], edges: Edge[]): DAGGraph

  // Kahn 算法拓扑排序，返回按层分组的节点 ID
  topologicalSort(graph: DAGGraph): string[][]

  // 按层执行：同层节点并发，层间串行
  async execute(workflow: WorkflowDefinition, onProgress: ProgressCallback): Promise<void>
}
```

## 五、文件操作方案

### 5.1 File System Access API 使用模式

```typescript
// 选择输出文件夹（用户授权后，句柄可存 IndexedDB 持久化）
const dirHandle = await window.showDirectoryPicker()

// 在文件夹内创建文件并写入
const fileHandle = await dirHandle.getFileHandle('kv_1.png', { create: true })
const writable = await fileHandle.createWritable()
await writable.write(blob)
await writable.close()

// 读取图片文件（用户选择）
const [fileHandle] = await window.showOpenFilePicker({
  types: [{ accept: { 'image/*': ['.png', '.jpg', '.jpeg', '.webp'] } }]
})
const file = await fileHandle.getFile()
```

### 5.2 输出文件夹权限持久化

用户首次选择输出文件夹后，将 `FileSystemDirectoryHandle` 存入 IndexedDB，下次打开应用自动请求权限恢复，无需重新选择。

### 5.3 工作流文件保存/加载

工作流 JSON 通过 `showSaveFilePicker` 保存为 `.json` 文件，通过 `showOpenFilePicker` 加载。
