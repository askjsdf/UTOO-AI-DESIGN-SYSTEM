# Phase 1 开发计划：工作流跑通

> 版本：v2.0（Web App 重构版）
> 日期：2026-04-09
> 变更说明：放弃 Electron，改为纯浏览器 Web App
> 关联文档：技术架构文档 v2.0 / 产品需求文档 v1.0
> 目标：实现可视化工作流搭建 + AI 管道执行 + 本地图片输出，验收通过后进入 Phase 2

---

## 一、Phase 1 目标与范围

### 目标

搭建工作流编排器的完整闭环：设计师在浏览器中打开页面，在可视化画布上拖拽节点、配置提示词、上传香水瓶图片，点击运行后系统自动调用 Gemini API 完成全套出图，13 张图片保存到本地文件夹。

### 范围内

- Vite + React Web App 项目搭建
- React Flow 工作流画布（5 种节点的 UI + 配置）
- 工作流 JSON 保存与加载（File System Access API）
- 内置 5 个工作流模板（4 个单类型 + 1 个全套）
- DAG 执行引擎（拓扑排序 + 节点调度）
- Gemini API 统一服务层
- 实时运行进度展示
- 输出图片写入本地文件夹（File System Access API）
- API Key 配置与测试

### 范围外（Phase 2 以后）

- tldraw 方案画布
- 批量处理与自动化
- 心跳监控
- 画布 ↔ 工作流双向联动

---

## 五、Phase 1 正式验收

### 5.1 验收前提条件

- Phase 1A 内部验收已通过
- Phase 1B 所有任务组已完成
- 准备好 3 张测试用香水瓶图片（覆盖不同风格：极简 / 奢华 / 几何）

---

### 5.2 验收测试操作流程

**测试由产品负责人亲自操作，开发者在旁观察记录，不主动引导。**

**Step 1：首次启动**
1. Chrome 浏览器打开 localhost:5173（或生产静态地址）
2. 进入设置页面，输入 Gemini API Key，点击「测试连接」
3. 看到「连接成功」提示
4. 点击「选择输出文件夹」，在系统弹窗中选择目标目录，授权访问

**Step 2：加载工作流模板**
1. 切换到「工作流」视图
2. 点击「加载模板」，选择「全套工作流」
3. 确认画布上出现完整的节点图（输入节点 → 分析节点 → 4 条并行生成分支 → 4 个输出节点）

**Step 3：上传测试图片并运行**
1. 点击画布上的「Image Input」节点
2. 在右侧配置面板点击「选择图片」，从系统弹窗选择第 1 张测试香水瓶图片
3. 点击顶部工具栏的「运行」按钮
4. 观察节点状态变化和底部进度条更新
5. 等待运行完成（预计 10–20 分钟）

**Step 4：查看输出结果**
1. 运行完成后查看进度区域的完成提示
2. 在本地文件管理器打开输出文件夹，确认文件结构和图片数量
3. 逐一查看 13 张图片

**Step 5：质量对比**
1. 同时打开原四个 AI Studio 原型（或其历史输出截图）
2. 对比相同输入图的输出结果
3. 对每类输出（KV / 矩阵 / 情绪板 / CMF）进行评分

**Step 6：重复测试**
1. 用第 2 张、第 3 张测试图重复 Step 3–5

---

### 5.3 验收标准

验收采用「强制项 + 评分项」两层结构：

#### 强制项（全部通过才能进入 Phase 2）

| 编号 | 验收项 | 判断方式 |
|------|--------|---------|
| M1 | 应用在 Chrome 中正常打开，无 JS 报错 | 控制台无红色错误 |
| M2 | 全套工作流模板加载后，节点图与架构文档描述一致 | 肉眼核对节点数量和连接关系 |
| M3 | 上传图片并点击运行后，节点状态实时变化（idle→running→completed） | 肉眼观察节点颜色变化 |
| M4 | 运行完成后，输出目录中存在完整的 13 张图片 | 打开文件管理器数图片数量 |
| M5 | 所有 13 张图片文件可正常打开，无损坏 | 双击预览每张图片 |
| M6 | 3 张测试图全部运行完成，没有出现整个工作流卡死 | 观察整个测试过程 |

#### 评分项（影响是否需要调整再进入 Phase 2）

每类输出（KV / 矩阵 / 情绪板 / CMF）独立评分，与原型输出对比：

| 分值 | 含义 |
|------|------|
| 3 分 | 输出质量明显优于原型 |
| 2 分 | 输出质量与原型持平 |
| 1 分 | 输出质量略低于原型，但可接受 |
| 0 分 | 输出质量不可接受，需要调整 |

**满分：** 4类 × 3张测试图 × 3分 = 36分
**通过线：** ≥ 24分（≥ 67%），且没有任何一类在 3 张测试图上全部得 0 分

---

### 5.4 验收结论与处理

| 结论 | 条件 | 后续行动 |
|------|------|---------|
| **通过** | 所有强制项通过 + 评分 ≥ 24 分 | 进入 Phase 2 开发 |
| **条件通过** | 所有强制项通过 + 评分 18–23 分 | 针对低分项调整提示词，一周内复验 |
| **不通过** | 任意强制项未通过，或评分 < 18 分 | 分析根因，修复后重新走完整验收流程 |
